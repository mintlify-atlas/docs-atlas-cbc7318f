title: "AI models"
description: "The AI models available for code generation in Nextly"
---

Nextly routes all AI calls through an [OpenRouter](https://openrouter.ai)-compatible endpoint. You choose a model when you start a generation; the model key is resolved to a full model ID at runtime.

## Available models

| Model Key | Model ID | Provider | Best For |
|---|---|---|---|
| `grok` | `x-ai/grok-4-fast:free` | xAI | Default during beta; free tier with generous limits |
| `codex` | `openai/gpt-5-codex` | OpenAI | Specialised code generation; highest accuracy on complex logic |
| `gemini` | `google/gemini-2.5-flash` | Google | Fast iteration; strong at large-context UI tasks |

### Grok (`x-ai/grok-4-fast:free`)

xAI Grok 4 Fast is the **default model during the Nextly beta**. It is served on the free tier, which means no API cost is passed to users on the hosted version. Grok is well-suited to full-page UI generation and iterative refinement.

### Codex (`openai/gpt-5-codex`)

OpenAI GPT-5 Codex is purpose-built for code. It produces clean, idiomatic TypeScript and handles complex component logic, custom hooks, and multi-file splits more reliably than general-purpose models.

### Gemini (`google/gemini-2.5-flash`)

Google Gemini 2.5 Flash is optimised for speed and large context windows. It is a strong choice when you need fast turnaround on prompts that reference many existing files or require broad layout changes.

## Secondary agents

After the main code agent finishes, two lightweight agents run to wrap up the generation:

- **Fragment title generator** — produces a short, 3-word title for the generated code fragment.
- **Response generator** — writes the friendly 1–3 sentence summary shown to the user.

Both agents use the model specified by the `OPENAI_FREE2_MODEL` environment variable, defaulting to `openai/gpt-oss-120b:free` when the variable is not set.

## Agent parameters

The main code agent runs with the following fixed parameters:

| Parameter | Value | Notes |
|---|---|---|
| `maxIter` | `15` | Maximum tool-call iterations before the network halts |
| `temperature` | `0.1` | Low temperature for deterministic, production-quality code output |
| Context window | Last 5 messages | Previous messages from the project are injected as prior context |

## Environment variables

| Variable | Purpose | Default |
|---|---|---|
| `OPENAI_API_KEY` | API key sent to the model endpoint | Required |
| `OPENAI_API_BASE` | Base URL for the OpenAI-compatible endpoint (e.g. OpenRouter) | Required |
| `OPENAI_FREE2_MODEL` | Model ID used by secondary agents | `openai/gpt-oss-120b:free` |

<Note>
  When self-hosting Nextly, you can swap to any OpenAI-compatible model by updating `OPENAI_API_BASE` to point at your preferred provider and setting `OPENAI_API_KEY` to the corresponding key. The model key you pass at generation time maps directly to the `modelMapping` object in `src/inngest/functions.ts` — update that mapping to add new model options.
</Note>
