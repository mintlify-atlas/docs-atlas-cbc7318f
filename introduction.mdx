Nextly takes a plain-language description of an app and returns working Next.js code with a live preview URL — no templates, no boilerplate, no manual scaffolding.

You describe what you want to build. Nextly spins up a sandboxed environment, runs an AI agent that writes and iterates on real code, and hands you a live preview alongside a downloadable codebase.

## What Nextly generates

Every project Nextly produces is a full Next.js 15 application with:

- **TypeScript** throughout — no implicit `any`, clean types
- **Tailwind CSS** for all styling — no plain CSS or external stylesheets
- **Shadcn UI** components pre-installed and used idiomatically
- App Router conventions, semantic HTML, and accessible markup by default

The agent writes real, production-quality files — not stubs or demos. It installs npm packages it needs, reads existing files before modifying them, and iterates until the app matches your description.

## How the stack fits together

| Layer | Technology | Role |
|---|---|---|
| Framework | Next.js 15 | App Router, server components, routing |
| Language | TypeScript | Type-safe code throughout |
| Styling | Tailwind CSS | Utility-first CSS |
| UI components | Shadcn UI | Pre-installed component library |
| Auth | Clerk | User accounts and session management |
| Background jobs | Inngest | Durable, event-driven generation pipeline |
| Code execution | E2B | Isolated sandbox where the agent writes and runs code |
| Database | PostgreSQL + Prisma | Projects, messages, and generated fragments |
